} @if (modalService.mode() === 'output-merging') {
This tool detects Output Merging by tracking a specific list of Poisoned Outputs. In Monero, every transaction creates outputs. To spend these, the owner must use them as inputs in a new transaction, hiding them among random "decoys" selected from the blockchain. Since decoys are chosen randomly from the entire blockchain history, the odds of a stranger accidentally selecting multiple outputs from your specific "poisoned" list to use as decoys in the same transaction are near zero. Therefore, if the tool observes a single transaction referencing multiple poisoned outputs simultaneously, it identifies that transaction as a Merge,confirming with high confidence that the entity controlling those funds is consolidating them.
How the tool works:
1. Initialization: For every initial transaction provided, the tool extracts all its outputs. These are marked as the initial set of "Poisoned Outputs." 2. Scanning: The tool retrieves every future transaction on the blockchain that references any of these poisoned outputs as a ring member. 3. Intersection Detection: It checks for overlaps. If a specific transaction hash appears in the lists of multiple different poisoned outputs, it is flagged as an Output Merging Transaction. 4. Recursive Tracing: The tool repeats this process recursively. The new outputs created by the detected Merging Transaction are treated as "poisoned," and the scan begins again for the next "round" (or hop). 5. Visualization: As soon as no further intersections are found, the process stops. The result displays the transaction hashes for each round, visualizing the flow of funds from the initial transactions (left) to the final consolidated outputs (right).
